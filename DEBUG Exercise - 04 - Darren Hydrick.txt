// This pseudocode segment is intended to compute and display
// the cost of home ownership for any number of users
// The program ends when a user enters 0 for the mortgage payment
start
   Declarations
      num mortgagePayment
      num utilities
      num taxes
      num upkeep
      num total
   startup()
   while mortgagePayment not equal to 0
      MainLoop()
   endwhile
   finishUp()
stop

startUp()
   output "Enter your mortgage payment or 0 to quit"
   input mtgPayment
return

mainLoop()
   output "Enter utilities"
   input utilities
   output "Enter taxes"
   input taxes
   output "Enter amount for upkeep"
   input upkeep
   total = mortgagePayment + utilities + taxes + upkeep
   output "Total is ", total
return

finishUp()
   output "End of program"
return

Problems and updated PSEUDOCODE:




1. Variable Name Mismatch - In the Declarations and the while loop the variable name mortgagePayment was used. However, inside the startUp() module, the input value was entered into a variable called mtgPayment.
	-- Updated the variables the same name everywhere.

2. The Infinite Loop - We get the mortgagePayment in the startUp() function, but we never asked for a new mortgage payment inside or at the end of the mainLoop().
	-- added an output and input to update the mortgagePayment so that the module would not run infinitely

3. Uninitialized Variable - Before when the program reached the line while mortgagePayment <> 0 for the very first time, there wasn't an assigned value to mortgagePayment yet, because of the naming mismatch mentioned in point #1. This could have caused unpredictable behavior.
